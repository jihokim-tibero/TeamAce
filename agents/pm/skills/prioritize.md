# 우선순위 산정 Skill

## 목적
기능/태스크의 우선순위를 데이터 기반으로 산정한다.

## 트리거
- 기능 목록이 3개 이상일 때
- 리소스 제약으로 범위 축소 필요 시

## 입력
- 기능 목록
- 비즈니스 목표/KPI
- 리소스 제약 (시간, 인력)

## 절차
1. RICE 프레임워크 적용:
   - Reach: 영향받는 사용자 수
   - Impact: 개인별 영향도 (3/2/1/0.5/0.25)
   - Confidence: 확신도 (100%/80%/50%)
   - Effort: 소요 인력·주
   - Score = (R × I × C) / E
2. MoSCoW 분류: Must / Should / Could / Won't
3. 결과 테이블 작성

## 출력 형식
| 기능ID | 기능명 | R | I | C | E | RICE Score | MoSCoW |

## 품질 체크리스트
- [ ] 모든 기능에 RICE 점수 산정
- [ ] Must 항목이 비즈니스 목표와 정렬
- [ ] Won't 항목에 제외 사유 명시

## 안티패턴
- 직감으로 "이게 중요하다" 판단
- 모든 기능을 Must로 분류
