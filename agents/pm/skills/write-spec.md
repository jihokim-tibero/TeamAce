# 기능정의서 작성 Skill

## 목적
기획서의 사용자 시나리오를 개발 가능한 기능 단위로 분해하고 명세한다.

## 트리거
- PRD 작성 완료 후
- 기존 기능 변경/추가 시

## 입력
- 완성된 기획서 (PRD)

## 절차
1. 기획서의 사용자 시나리오에서 기능 단위 추출
2. 기능 목록 테이블 작성 (기능ID, 기능명, 설명, 우선순위)
3. 각 기능 상세 명세:
   - 입력 조건 (데이터 필드명 camelCase)
   - 처리 로직 (비즈니스 규칙, 분기 조건 명시)
   - 출력 결과 (데이터 필드명 camelCase)
   - 예외 처리 (에러 코드 + 사용자 표시)
   - 상태 전이 (해당 시)
4. 수락 기준 작성 — Given-When-Then 형태
5. contracts/pm-to-ux.md, pm-to-be.md 체크리스트 대조

## 출력 형식
agents/pm.md의 기능정의서 형식 참조

## 품질 체크리스트
- [ ] PRD 시나리오 대비 기능 매핑률 100%
- [ ] 기능당 입력/처리/출력/예외 4항목 모두 정의
- [ ] 필드명 camelCase 준수
- [ ] 수락 기준이 Given-When-Then 변환 가능
- [ ] 에러 코드가 구체적으로 정의됨

## 안티패턴
- "처리한다"만 쓰고 구체적 로직 미명시
- 필드명 snake_case 또는 한글 혼용
- 에러 처리 항목 누락
