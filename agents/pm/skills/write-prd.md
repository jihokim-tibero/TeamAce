# PRD(기획서) 작성 Skill

## 목적
사용자 요구사항을 데이터 드리븐 기획서로 변환한다.

## 트리거
- 새 기능/서비스 기획 요청
- 기존 기능 개선 기획 요청

## 입력
- 사용자 요구사항 (구두 또는 문서)
- 기존 데이터/지표 (있는 경우)
- 경쟁사/시장 정보 (있는 경우)

## 절차
1. 요구사항 명확화 — 모호한 부분 질문으로 해소
2. 가설 수립 — "~하면 ~할 것이다" 형태
3. 배경/목적 작성 — 데이터 근거 포함
4. 목표 지표(KPI) 정의 — 현재값 → 목표값, 측정 방법
5. 범위 정의 — In Scope / Out of Scope
6. 타깃 사용자 정의 — 세그먼트 + 핵심 니즈
7. 사용자 시나리오 — 주요 흐름 + 예외 흐름 (최소 2개)
8. 의존성 파악
9. Agent 인터페이스 작성 — UX/FE/BE/QA 각각에 구체적 지시
10. 성공 판정 기준 정의

## 출력 형식
CLAUDE.md 또는 agents/pm.md의 기획서 형식 참조

## 품질 체크리스트
- [ ] 가설이 검증 가능한 형태인가
- [ ] 모든 KPI에 수치 기준이 있는가
- [ ] In/Out Scope가 명확한가
- [ ] 사용자 시나리오에 예외 흐름이 포함되었는가
- [ ] Agent 인터페이스 4개 에이전트 모두 작성했는가
- [ ] 모호 표현("적절히", "필요 시")이 없는가

## 안티패턴
- 데이터/근거 없이 직감으로 기획
- "~하면 좋겠다" 식의 비검증 가능한 목표
- Agent 인터페이스 섹션 누락
