# PM → BE 핸드오프 계약

## PM이 BE에게 전달해야 하는 필수 산출물

### 1. 기능정의서
- [ ] 기능 목록 테이블 (기능ID, 기능명, 우선순위)
- [ ] 각 기능의 비즈니스 규칙 (처리 로직)
- [ ] 데이터 요구사항 (어떤 데이터를 저장/조회/수정/삭제하는가)
- [ ] 상태 전이 정의 (있는 경우)
- [ ] 외부 시스템 연동 요구 (있는 경우)

### 2. 성능 기준
- [ ] 목표 응답 시간 (기본: P95 < 500ms)
- [ ] 예상 동시 사용자 수 (알려진 경우)
- [ ] 데이터 볼륨 예측 (알려진 경우)

### 3. 보안 요구사항
- [ ] 인증/인가 방식 명시
- [ ] 민감 데이터 처리 요구 (있는 경우)

## 필드 네이밍
- 기능정의서의 필드명 = BE API 응답 필드명 = DB 컬럼명의 camelCase 변환
- BE가 필드명을 변경해야 하는 경우, PM에게 기능정의서 업데이트 요청 필수

## 산출물 위치
- 기능정의서: GitHub → **Notion** (Notion MCP) / GitLab → **Git Wiki** (`glab wiki`)
- BE Agent는 Notion 또는 Git Wiki에서 기능정의서를 조회

## 검증 기준
- BE Agent가 Notion/Wiki의 기능정의서만 보고 API 엔드포인트 목록과 DB 스키마 초안을 도출 가능해야 함
- 비즈니스 규칙에 모호함이 없어야 함 ("경우에 따라" → 구체적 조건 분기)
