# 에이전트 산출물 평가 기준

## 목적

에이전트의 산출물 품질을 **정량적으로 측정**하여,
모델 교체/컨텍스트 변경 시에도 일관된 품질을 유지하는지 판단한다.

---

## 평가 체계

각 산출물을 0-10점으로 평가. **7점 이상이 합격 기준.**

---

## PM Agent 산출물 평가

### 기획서 (PRD)

| 평가 항목 | 배점 | 기준 |
|-----------|------|------|
| 완성도 | 3 | 필수 섹션 6개 중 존재 비율 × 3 |
| 구체성 | 3 | 모호 표현 0개 = 3, 1-2개 = 2, 3개+ = 1 |
| 측정 가능성 | 2 | KPI 중 정량 기준이 있는 비율 × 2 |
| Agent 인터페이스 | 2 | 4개 에이전트(PUB/FE/BE/QA) 지시 존재 비율 × 2 |

### 기능 명세서

| 평가 항목 | 배점 | 기준 |
|-----------|------|------|
| 기능 커버리지 | 3 | PRD 시나리오 대비 기능 매핑률 × 3 |
| 명세 완성도 | 3 | 기능당 입력/처리/출력/예외 4항목 존재율 × 3 |
| 수락 기준 | 2 | Given-When-Then 변환 가능한 기준 비율 × 2 |
| 네이밍 일관성 | 2 | camelCase 준수율 × 2 |

## PUB Agent 산출물 평가

### View 컴포넌트 코드

| 평가 항목 | 배점 | 기준 |
|-----------|------|------|
| 화면 커버리지 | 1 | 기능 명세서 기능 대비 View 매핑률 × 1 |
| 상태 구현 | 2 | 인터랙티브 컴포넌트의 5 상태 코드 구현율 × 2 |
| data-testid | 1 | 인터랙티브 요소의 testid 코드 구현율 × 1 |
| Impeccable 준수 | 2 | 8개 안티패턴 미위반율 × 2 (전체 통과 = 2) |
| 접근성 코드 | 1 | ARIA + 키보드 + 색 대비 + reduced-motion 구현 |
| View 순수성 | 1 | API 호출/store import 부재 (위반 0 = 1, 1+ = 0) |
| 디자인 토큰 | 1 | OKLCH 색상 + 타이포 + 간격 + 반경 정의 완성도 |
| 네이밍 일치 | 1 | PM 기능 명세서 필드명과 일치율 |

| 평가 항목 | 배점 | 기준 |
|-----------|------|------|
| 타입 안전성 | 1 | `any` 타입 0개 = 1, 1개+ = 0 |

| View 미수정 | 1 | PUB 소유 파일 변경 0 = 1, 1+ = 0 |

## 사용 방법

1. 에이전트 산출물 완료 시 리드가 이 기준으로 평가
2. 7점 미만 시 미달 항목을 구체적으로 피드백하고 보완 요청
3. 모델 교체 후 동일 입력에 대해 재평가하여 품질 편차 측정
4. 점수 이력을 기록하여 에이전트 성능 추이 추적
