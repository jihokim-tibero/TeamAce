# PUB 핵심 설계 원칙

작업 시작 전 읽고, 모든 작업 과정에서 준수한다. 완료 시 산출물 대상으로 재확인한다.

---

## 원칙

### P1. 코드가 곧 시안
View 컴포넌트(TSX + CSS)가 최종 디자인 산출물이다.
Figma나 별도 시안 없이 코드만으로 디자인 의도가 완전히 전달되어야 한다.

### P2. Impeccable 8대 안티패턴 위반 제로
Inter 기본값, 순수 검정(#000), 보라색 그라데이션 기본값, 카드 안의 카드,
회색 텍스트 on 컬러 배경, 장식용 보더, 동일 간격 반복, 기본 그림자 남발.
하나라도 위반하면 실패.

### P3. 5 상태 완전 구현
모든 인터랙티브 컴포넌트에 idle/loading/success/error/empty 5 상태가
코드로 구현되어 있어야 한다. 누락 상태가 있으면 실패.

### P4. View 순수성
View 컴포넌트에 API 호출, fetch, store import, 전역 상태 직접 접근이 없어야 한다.
Props + 콜백으로만 외부와 통신한다. 위반 시 실패.

### P5. 접근성 내장
OKLCH 색상 체계, WCAG AA 색 대비, ARIA 레이블, 키보드 네비게이션,
prefers-reduced-motion 대응이 코드에 구현되어 있어야 한다.

### P6. data-testid 완전성
모든 인터랙티브 요소에 `[컴포넌트]-[역할]` 형식의 data-testid가 있어야 한다.
QA 자동화 테스트가 이 ID만으로 요소를 선택할 수 있어야 한다.

### P7. 코드 위생
- `any` 타입 사용 금지 — Props 인터페이스 명시
- 파일당 300줄 이하
