# 스냅샷 테스트 Skill

## 목적
UI와 API 응답의 의도하지 않은 변경을 감지한다.

## 트리거
- FE 컴포넌트 변경 후
- BE API 응답 구조 변경 후

## 입력
- 변경된 컴포넌트 / API 엔드포인트
- 기존 스냅샷 베이스라인

## 절차
### UI 스냅샷
1. 각 컴포넌트의 5 상태별 렌더링 스냅샷 생성
2. 기존 스냅샷과 비교 (diff)
3. 판정:
   - diff 없음 → PASS
   - 의도된 변경 → 베이스라인 업데이트 (리드 승인)
   - 의도하지 않은 변경 → FAIL (버그 리포트)

### API 응답 스냅샷
1. 주요 엔드포인트의 성공/에러 응답 스냅샷 생성
2. 기존 스냅샷과 비교:
   - 필드 추가 → 허용 (하위 호환)
   - 필드 삭제/타입 변경 → 차단 (비호환 변경)
3. 비호환 변경 발견 시 BE에 수정 요청

## 출력 형식
```markdown
## 스냅샷 테스트 결과
**실행일**: YYYY-MM-DD

### UI 스냅샷
| 컴포넌트 | 상태 | 결과 | diff |
|---------|------|------|------|

### API 스냅샷
| 엔드포인트 | 결과 | 변경 유형 |
|-----------|------|-----------|

### 판정
- UI: [N/N 통과]
- API: [N/N 통과]
- 비호환 변경: [있음/없음]
```

## 품질 체크리스트
- [ ] 모든 변경 컴포넌트 5 상태 스냅샷 확인
- [ ] 모든 변경 API 응답 스냅샷 확인
- [ ] 비호환 변경 감지 시 차단 + 리포트
- [ ] 의도된 변경은 베이스라인 업데이트

## 안티패턴
- 스냅샷 diff를 확인 없이 전체 업데이트
- API 응답 필드 삭제를 "마이너 변경"으로 무시
- 스냅샷 베이스라인 업데이트 시 변경 이유 미기록
