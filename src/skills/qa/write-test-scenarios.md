# 테스트 시나리오 작성 Skill

## 목적
리스크 기반으로 우선순위가 부여된 테스트 시나리오를 작성한다.

## 트리거
- PM 기획서 + PUB View 명세 + FE/BE 코드가 준비된 후

## 입력 (블랙박스 근거 — 소스 내부 금지)
- PM 기획서 (수락 기준)
- PM 기능 명세서 (기능 목록, 상세 명세)
- PUB View 명세 (data-testid, 상태 정의, 사용자 여정)
- BE API 명세 (엔드포인트, 에러 코드, 응답 스키마)
- FE/BE 코드 변경 diff — **파일·범위 식별 목적만**. 내부 구현 로직을 읽어 테스트를 맞추면 안 된다.

## 절차
1. 수락 기준에서 Given-When-Then TC 도출
2. 리스크 분석:
   - Critical: 사용자 핵심 가치 경로, 결제/인증, 데이터 손실 가능성
   - High: 주요 기능의 예외 흐름, 에러 처리
   - Medium: 부가 기능, UI/UX 세부사항
   - Low: 엣지 케이스, 미약한 영향
3. 각 TC에 리스크 근거 명시 ("왜 이 우선순위인가")
4. data-testid 기반 자동화 가능 여부 판단
5. 검증 유형 분류 (기능/UI/성능/접근성/보안/회귀)

## 출력 형식
```markdown
## TC-[번호]: [시나리오명]
**Given**: [초기 상태]
**When**: [액션] (data-testid: `[id]`)
**Then**: [기대 결과]

**우선순위**: Critical / High / Medium / Low
**리스크 근거**: [비즈니스 이유]
**자동화 가능**: Yes / No
**검증 유형**: 기능 / UI / 성능 / 접근성 / 보안 / 회귀
```

## 품질 체크리스트
- [ ] 수락 기준 대비 TC 매핑률 100%
- [ ] 모든 TC에 리스크 근거 존재
- [ ] Critical TC가 핵심 가치 경로를 커버
- [ ] data-testid 기반 자동화 가능 TC ≥ 90%
- [ ] 에러/예외 흐름 TC 존재

## 범용 테스트 패턴

### 크로스 레퍼런스 검증
기능 명세서와 디자인 명세서를 교차 대조하여 누락된 테스트 갭을 식별한다.
한쪽에만 있는 기능이 테스트에서 빠지지 않도록 매핑 테이블을 작성한다.

### 스트리밍 프로토콜 검증
SSE/WebSocket 등 스트리밍 API는 이벤트 순서와 타입 존재/부재를 모드·상태별로 검증한다.
다단계 인터랙티브 흐름에서는 중간 응답의 식별자를 추출하여 후속 요청에 재사용하는 경로를 테스트한다.

### 권한 기반 접근 제어 검증
비권한 토큰으로 권한 필요 작업을 시도하여 HTTP 403 거부를 확인한다.
정상 경로와 별도의 TC로 분리한다.

### 도메인 에러 코드 분리
도메인별 에러 코드는 범용 에러 시나리오와 번들하지 않고 전용 TC로 분리한다.
에러 코드별 응답 본문 구조와 메시지를 개별 검증한다.

### 모바일 UI 세분화 검증
모바일 UI는 세분화된 검증 포인트로 분리한다: input mode 속성, 터치 타겟 크기(≥44px), 키보드 가림 방지.

### 성능 벤치마크 정렬
성능 기준(P95 응답시간 등)은 PM 기능 명세서의 KPI와 정렬한다.
API 레벨 측정과 E2E 측정을 혼동하지 않는다.

## 명세 ↔ 실제 동작 불일치 시 (필수 절차)
1. 테스트를 현재 동작에 맞게 수정하지 않는다.
2. 불일치 유형 판단:
   - 동작이 명세에 반함 → **버그 리포트** (담당: PUB/FE/BE)
   - 명세가 모호·누락·모순 → **명세 정정 요청** (담당: PM, 관련 개발 참조)
3. 해당 TC는 **Blocked** 상태로 표시하고, 합의 결과가 반영된 뒤에만 갱신한다.

## 안티패턴
- 리스크 근거 없이 "중요해 보여서" Critical 배정
- happy path만 테스트하고 에러 케이스 무시
- data-testid 미활용 (CSS selector 기반 TC)
- **소스 코드 내부 로직을 읽고 그에 맞춰 TC를 작성** — 블랙박스 원칙 위반
- **명세와 동작이 다를 때 테스트를 현재 동작에 맞춰 수정** — P8 위반. 버그 또는 명세 정정 이슈로 제기할 것
